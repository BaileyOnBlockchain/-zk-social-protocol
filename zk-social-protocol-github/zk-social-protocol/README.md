# ZK Social Protocol — Smart Contracts

A suite of Solidity smart contracts for a privacy-first, decentralised social protocol. Built on ZK proof primitives — users can post, join groups, tip, stake, and govern anonymously without sacrificing verifiability.

---

## Overview

This contract suite enables:

- **Anonymous posting** verified on-chain via Semaphore ZK proofs — identity hidden, authenticity provable
- **ZK group membership** — join private groups and prove membership without revealing who you are
- **AI-assisted moderation** — optional EZKL on-chain toxicity verification via ZK machine learning proofs
- **Token-gated access** — ERC20 or ERC721 balance checks for channel/community access
- **On-chain scam reporting** — community trust scoring with verifiable evidence
- **Profile NFTs** — one-per-address soulbound-style identity layer with IPFS metadata
- **Staking vault** — Yearn-style APY staking on the native token
- **Multi-sig treasury** — M-of-N governance with timelock for protocol funds

## Contracts

### `PrivNet.sol`
Core anonymous posting contract. Posts are content-hashed and linked to a Semaphore nullifier — provably unique per identity, fully anonymous. Supports optional verifiable author mode for users who choose to dox themselves. Tips accepted per post in native token.

### `PrivNetOptimized.sol`
Gas-optimised variant targeting ZKsync Era. Struct packing reduces storage slots: `timestamp` + `totalTips` packed into one slot, `bool verified` packed alongside. Uses `immutable` for Semaphore address to save ~2,100 gas per read.

### `PrivNetSemaphore.sol`
Extended version integrating EZKL ZK machine learning proof verification. Before a post is accepted, an optional toxicity check runs — the poster submits a ZK proof that their content scores below a configurable threshold. Threshold and moderation toggle are owner-adjustable.

### `GroupManager.sol`
Private group creation and management with Semaphore integration. Each group gets a Semaphore group ID; metadata (name, description, encryption flag, ZK access flag) stored on-chain. Groups can be encrypted or ZK-access-only.

### `TokenGatedChannel.sol`
ERC20 or ERC721 balance-gated channel access. Supports minimum token balance for ERC20, NFT ownership checks for ERC721, optional whitelist per channel, and admin-controlled access grants. Multiple channels with independent gating rules.

### `ScamReports.sol`
Decentralised on-chain scam reporting. Reporters submit evidence against wallet addresses; reports can be verified or dismissed by owner. Trust scores (0–100) are computed per wallet from report counts and severity — lower score = more suspicious.

### `ProfileNFT.sol`
ERC721 profile NFT with one-per-address enforcement. Metadata stored on IPFS, URI updatable by token holder. Inherits ERC721URIStorage + ERC721Enumerable with proper override resolution.

### `PrivToken.sol`
ERC20 utility token for tipping, staking, and governance. 1B hard cap, per-address mint cap for anti-sybil. Supports batch minting with reason tracking. Burnable.

### `Treasury.sol`
M-of-N multi-signature treasury with timelock. Proposals require threshold signatures + delay before execution. Auto-executes on final approval if timelock has passed. Emergency withdrawal available for non-native tokens only. Packed structs for gas efficiency.

### `StakingVault.sol`
Yearn-style staking vault. Users deposit the protocol token and earn APY from platform fees. Configurable APY in basis points. Standard stake/unstake/claim flow with per-user accrual tracking.

### `IEZKLVerifier.sol`
Interface for the EZKL ZK proof verifier. Accepts a raw proof and public inputs (signal hash + toxicity score) and returns a boolean. Replace with deployed EZKL verifier address after circuit setup.

### `Semaphore.sol`
Full Semaphore protocol implementation. Group admins create and manage groups; members generate ZK proofs of membership and signal anonymously. Merkle root duration tracking for proof validity windows.

### `SemaphoreVerifier.sol`
Groth16 verifier contract generated by snarkJS from the Semaphore circuit. Contains hardcoded verification key points. Used by `Semaphore.sol` for on-chain proof verification.

## Security Notes

- All state-changing functions are `nonReentrant` where ETH or token transfers occur
- Treasury proposals enforce timelock even for single-signer setups — no instant execution
- Semaphore nullifier hashes prevent double-posting / double-voting per identity
- EZKL verifier is a placeholder interface — deploy actual circuit verifier before enabling moderation
- `emergencyWithdraw` in Treasury explicitly blocks withdrawing the native gov token — must use proposals
